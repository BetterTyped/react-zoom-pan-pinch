# useControls

Access the **transform handlers** from any component inside `TransformWrapper`
— no render props needed. Returns the same API you get from render props:
`zoomIn`, `zoomOut`, `resetTransform`, `centerView`, `zoomToElement`, and
`setTransform`, plus the raw `instance` ref.

## Signature

```ts
function useControls(): ReactZoomPanPinchContentRef
```

## Return Value

```ts
{
  instance: ReactZoomPanPinchContext;  // raw library instance
  zoomIn: (step?, animationTime?, animationName?) => void;
  zoomOut: (step?, animationTime?, animationName?) => void;
  resetTransform: (animationTime?, animationName?) => void;
  centerView: (scale?, animationTime?, animationName?) => void;
  zoomToElement: (node, scale?, animationTime?, animationName?) => void;
  setTransform: (x, y, scale, animationTime?, animationName?) => void;
}
```

## When to Use

- Building a **toolbar or control panel** separate from the content area
- Triggering zoom/pan from **buttons, menus, or keyboard shortcuts**
- Any descendant component that needs to **control** the transform but not read its state

For reading transform state reactively, use `useTransformComponent` or
`useTransformEffect` instead.

## Example

```tsx
import { TransformWrapper, TransformComponent } from "react-zoom-pan-pinch";
import { useControls } from "react-zoom-pan-pinch";

function Toolbar() {
  const { zoomIn, zoomOut, resetTransform, centerView } = useControls();

  return (
    <div className="toolbar">
      <button onClick={() => zoomIn()}>Zoom In</button>
      <button onClick={() => zoomOut()}>Zoom Out</button>
      <button onClick={() => resetTransform()}>Reset</button>
      <button onClick={() => centerView()}>Center</button>
    </div>
  );
}

function App() {
  return (
    <TransformWrapper>
      <Toolbar />
      <TransformComponent>
        <img src="map.jpg" alt="Map" />
      </TransformComponent>
    </TransformWrapper>
  );
}
```

## vs Render Props

Render props work well for simple cases, but `useControls` is cleaner when:

- The controls live in a **deeply nested** or **separate** component
- You need transform actions in a **custom hook**
- You want to avoid **prop drilling** through multiple layers

Both patterns provide the same underlying API — `useControls` is just syntactic
sugar over `useTransformContext` + `getControls`.

## Requirements

Must be called inside a component that is a **descendant** of
`TransformWrapper`. Throws an error if used outside the transform context.
