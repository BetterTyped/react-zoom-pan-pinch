# useTransformComponent

Subscribe to every transform change and **return a value** — typically JSX —
that updates in real time. This hook is the primary way to build UI that
reflects the current zoom, pan, or scale state without manual event wiring.

## Signature

```ts
function useTransformComponent<T>(
  callback: (state: ReactZoomPanPinchContextState) => T,
): T
```

**`state`** contains:

```ts
{
  state: {
    previousScale: number;
    scale: number;
    positionX: number;
    positionY: number;
  };
  instance: ReactZoomPanPinchContext;
}
```

## How It Works

Internally, `useTransformComponent` subscribes to the library's `onChange`
event and calls your callback on every transform update. The return value is
stored in React state (`useState`), so the consuming component re-renders
whenever the transform changes.

## When to Use

- **Scale badges** — show the current zoom level (`state.scale`)
- **Coordinate displays** — show positionX / positionY
- **Conditional rendering** — show/hide elements based on zoom level
- Any UI element that should **visually update** on every transform tick

## Examples

### Scale Indicator

```tsx
function ScaleBadge() {
  return useTransformComponent(({ state }) => (
    <div className="badge">{state.scale.toFixed(2)}x</div>
  ));
}
```

### Zoom Percentage

```tsx
function ZoomPercent() {
  return useTransformComponent(({ state }) => (
    <span>{Math.round(state.scale * 100)}%</span>
  ));
}
```

### Coordinates

```tsx
function Coords() {
  return useTransformComponent(({ state }) => (
    <div>
      X: {state.positionX.toFixed(0)}, Y: {state.positionY.toFixed(0)}
    </div>
  ));
}
```

## Performance

The callback fires on **every** transform event — every mouse move during a
pan, every wheel tick, every pinch frame. Keep the callback lightweight.
Avoid expensive computations or large JSX trees inside it. If you only need
to run side-effects (not render UI), use `useTransformEffect` instead.

## vs Other Hooks

| Hook | Purpose | Re-renders? |
|------|---------|-------------|
| `useTransformComponent` | Render UI from state | Yes |
| `useTransformEffect` | Side-effects on change | No |
| `useControls` | Trigger actions | No |
| `useTransformContext` | Raw instance access | No |

## Requirements

Must be called inside a descendant of `TransformWrapper`.
