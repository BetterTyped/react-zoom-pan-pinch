# Initial transform

Set the starting view with **`initialScale`**, **`initialPositionX`**, and
**`initialPositionY`**. Positions are in the same **pixel space** as the
transform state (`positionX` / `positionY`).

If **`centerOnInit`** is `true` (the default in many apps), the library will
**center** the content after mount and can override your initial translation.
This story sets **`centerOnInit={false}`** so the initial scale and position
stay exactly as configured.

Try **Reset** or **Center** from the control bar to compare the default
behavior helpers with your chosen initial view.
