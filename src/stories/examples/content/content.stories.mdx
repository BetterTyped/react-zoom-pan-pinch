# Mixed Content

Real-world zoomable interfaces rarely contain a single image—they hold
**dashboards, text, charts, buttons, scrollable lists**, and other interactive
widgets. This story proves the library handles them all, including **gesture
exclusion zones** that let parts of the content opt out of specific
interactions.

## Why exclusion zones matter

When your transformed content contains scrollable areas, clickable controls,
or embedded widgets, you often need to **prevent certain gestures from
reaching the library** on specific elements. Without exclusion, a user trying
to scroll a list would instead pan the entire canvas—or a pinch to zoom an
embedded map would zoom the outer viewport.

The library solves this with the **`excluded` array** available on `panning`,
`wheel`, `pinch`, and `doubleClick` options. Pass one or more **CSS class
names**; any event that originates on (or inside) an element carrying that
class is silently ignored by that gesture handler.

## How it works

```tsx
<TransformWrapper
  panning={{ excluded: ["panningDisabled"] }}
  wheel={{ excluded: ["wheelDisabled"] }}
  pinch={{ excluded: ["pinchDisabled"] }}
>
  <TransformComponent>
    <div className="panningDisabled">
      {/* Drag here does NOT pan the canvas */}
      <ScrollableList />
    </div>
    <div className="wheelDisabled">
      {/* Mouse wheel here does NOT zoom */}
      <SomeWidget />
    </div>
    <div className="pinchDisabled">
      {/* Touch pinch here does NOT zoom */}
      <EmbeddedMap />
    </div>
    <p>Normal content — all gestures work here</p>
  </TransformComponent>
</TransformWrapper>
```

| Prop | Gesture blocked | Typical use case |
|---|---|---|
| `panning.excluded` | Click-and-drag | Scrollable lists, text selection, draggable widgets |
| `wheel.excluded` | Mouse wheel / trackpad scroll | Dropdowns, inner scroll containers, code editors |
| `pinch.excluded` | Two-finger pinch | Embedded maps, multi-touch canvases, sliders |
| `doubleClick.excluded` | Double-click zoom | Editable text, buttons that need double-click |

> **No JavaScript wiring required.** Add the CSS class to the element (or a
> parent), and the library reads it from the event target's class list at
> interaction time. Classes are checked via `classList.contains()`, so they
> work on any depth of nesting.

## Combining exclusions

You can stack multiple exclusion classes on a single element to block several
gestures at once:

```tsx
<div className="panningDisabled wheelDisabled">
  {/* Both panning AND wheel zoom are blocked here */}
</div>
```

The exclusion is **per-gesture**, so unrelated gestures still pass through.
For example, an element with `panningDisabled` still responds to wheel zoom
and pinch zoom.

## Common patterns

**Scrollable sidebar inside a zoomable canvas** — wrap the sidebar in a
`panningDisabled wheelDisabled` container so vertical scrolling and wheel
events stay local.

**Interactive form controls** — add `panningDisabled` to the form wrapper so
users can click inputs, drag sliders, and select text without moving the
canvas.

**Nested zoomable region** — add `pinchDisabled wheelDisabled` to the inner
region and attach a second `TransformWrapper` inside it for independent zoom
control.
